# ファイル07：補助金審査・交付事務局運営
## 住宅用エネルギーシステム・省エネ設備導入支援事業

> **適用業務例**：令和8年度福岡市住宅用エネルギーシステム導入支援事業補助金交付業務委託、令和8年度事業所の省エネ設備導入支援事業補助金審査事務等業務委託  
> **契約方式**：単価契約（件数ベース）

---

## 1. このカテゴリの業務特性

環境計量系業務とは**全く異なる業務種別**。
- 物理的な測定・分析は一切なし
- **件数ベースの単価契約**が標準
- 補助金の交付（振込）まで行う場合は**資金管理リスク**がある
- 個人情報・事業者情報を大量に扱うため、**情報セキュリティ要件が特に厳格**
- ダブルチェック（2名審査）が必須要件

---

## 2. 業務の2層構造

```
業務全体
  ├── (1) 事務局運営業務（1式）
  │       ├── 問い合わせ対応（電話・FAX・メール）
  │       ├── 定期打ち合わせ・報告
  │       ├── 月次報告（毎月）
  │       ├── アンケート送付・回収・集計（住宅系のみ）
  │       └── 電力使用状況報告書の送付・集計（住宅系のみ）
  │
  └── (2) 補助金審査業務（単価×件数）
          ├── A: 交付対象申請にかかる業務（○件）
          └── B: 交付請求にかかる業務（○件）
```

## 4. 積算の重要ポイント

### 4.1 単価項目の特定

補助金審査業務は**単価契約**のため、以下を必ず特定する。

```
単価対象の確認事項:
  □ 交付対象申請の件数（○件）→ 申請審査の単価×件数
  □ 交付請求の件数（○件）→ 請求審査の単価×件数
  □ 事務局運営（1式）→ 月額または期間一括
  □ 振込手数料の負担（受託者負担か発注者負担か）
  □ アンケートの印刷・郵送費（部数×単価）
```

### 4.2 振込手数料の注意

「振込手数料は補助金審査・交付業務の単価に含む」と明記される場合がある。
1件あたりの振込手数料を見積に含めること（銀行・ゆうちょ・ネットバンク等で異なる）。

### 4.3 補助金の立替リスク

住宅用エネルギー系業務では、**受託者が補助金を申請者に交付し、後から協議会から回収する**スキームが採用されることがある。
この場合、**補助金の立替資金が必要**となるため、資金調達コストを考慮すること。

## 5. 支払い条件・報告スケジュール

### 住宅用エネルギーシステム（例）

| 報告 | 対象期間 | 報告期限 |
|------|---------|---------|
| 前期（事務局） | 4月〜9月末 | 10月5日 |
| 後期（事務局） | 10月〜3月末 | 3月31日 |
| 第1回（審査） | 4月〜9月末 | 10月5日 |
| 第2回（審査） | 10月〜11月末 | 12月7日 |
| 第3回（審査） | 12月 | 1月5日 |
| 第4回（審査） | 1月 | 2月5日 |
| 第5回（審査） | 2月 | 3月5日 |
| 第6回（審査） | 3月 | 3月31日 |

> **注意**：後半（12月以降）は月単位の細かい報告が求められる。業務量の繁閑を考慮した人員計画が必要。

## 6. 成果物要件

| 成果物 | 内容 |
|--------|------|
| 月次報告（様式1） | 毎月末時点の申請状況集計 |
| 業務報告（様式2） | 報告期間ごとの審査件数・交付件数等 |
| 一覧表データ | 申請者情報・審査状況・振込記録（Excel） |
| 年間報告書 | 事業概要・実績・課題・アンケート集計結果等 |
| 電子データ | PDF・Word（またはExcel）1式 |

## 7. 情報セキュリティ要件（重要）

```
確認事項:
  □ 「個人情報・情報資産取扱特記事項」への準拠が求められているか
  □ 氏名・住所・生年月日・電話番号の管理精度要件
  □ 情報漏洩時の報告義務
  □ 委託者から貸与される物品・資料の管理
  □ 業務遂行責任者の設置（省エネ設備系では明示）
  □ 著作権の帰属（発注者・協議会事務局帰属が標準）
```
